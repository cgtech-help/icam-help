Post-Processor Minor Words

Minor words are post-processor command keywords that are represented in the CLDATA using unique integer codes of the 2000 class record data. These words must appear in an APT source file after the / character. The tables below list the built-in Minor words supported by Icam Post, sorted both alphabetically and numerically by integer code.

New Minor words and their associated codes can be added to the default words file, or to the words file associated with an interface kit. Minor word names should (by convention) start with an alphabetic character and consist only of alphanumeric characters, to a maximum length of 24 characters.

Minor word codes must be positive whole numbers. To avoid defining a new code for your own use that might ultimately conflict with a built-in code, use the range 900–999.
