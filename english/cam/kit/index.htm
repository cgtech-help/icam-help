CAM-POST Interface Kits

CAM-POST can interface with a wide variety of CAM systems through the use of CAM Interface Kits. The purpose of a CAM interface kit is to adjust configuration settings so that CAM-POST can recognize the physical CL file format of the CAM system. Some CAM systems have an inflexible post-processor command syntax. In this case, the interface kit will also include macros that transform these non-conforming post-processor commands to the format that CAM-POST requires.

Selecting Interface Kits

Automatic

GENER choses an appropriate interface kit based on the input CL file type and content.

Creating your Own Custom Interface Kits

To create a custom kit, it is recommended to modify an existing interface kit by copying a kit file from the installation kit subdirectory to the application data kit subdirector. Once the copy is done, make any modifications necessary to the kit files in the application data area. You may choose to leave the name as-is, or to modify the application data kit name to something unique (do not include spaces in the name).
