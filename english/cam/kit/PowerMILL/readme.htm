PowerMill TO CAM-POST INTERFACE DOCUMENTATION

Overview

The PowerMill Interface Kit is designed to accommodate differences in sytnax between PowerMill's and ICAM's post processing languages. The Interface Kit supports ASCII.cut cutter location (CLDATA) files generated. The binary cut file format is not supported. The PowerMill CLDATA file output type and format can be selected from the NC Programs tab of the Options dialog box, as shown below:

Post-processor Commands Conversion

Some of the post-processor commands output by PowerMill do not match the input format requirements of CAM-POST. A description of the PowerMill CLDATA file format can be found in the cldata.htm file in the PowerMill Interface Kit folder. The CAM-POST post-processor commands are documented in the "Post-Processor Vocabulary" chapter of the Users Guide.

The PowerMill Interface Kit includes a kit.mac file containing macros that, as much as possible, convert PowerMill post processor commands to CAM-POST format. These macros are by default matched after any macros defined in the post processor, so their effects can be easily overridden.
