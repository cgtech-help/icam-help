GNC TO CAM-POST INTERFACE DOCUMENTATION

Overview

The GNC Interface Kit is designed to accommodate differences in syntax between
GNC and ICAM's postprocessing languages. GNC's APT commands in a cutter
location source (.CL) file are input to CAD-APT and/or CAM-POST. The syntax for
these statements are found in the "GNC CLdata, Information for Postprocessor
Writers" manual. See the CAM-POST chapter "Post-processor Vocabulary" for more
information on CAM-POST supported syntax.

Only automatically generated GNC commands are accounted for in the GNC
Interface Kit. Other postprocessor statements are passed through. Not all
translations shown in this document may be used with all ICAM posts. For
example, a CYCLE statement may be output by GNC and correctly reformatted by
the interface, but the post-processor may not support that CYCLE type; a
postprocessing error would result.

A "*.CL" file will be posted regardless of the number of translations that
cannot be made. The messages in the "*.lis" and "*.lst" files are meant to
warn users that there are items in the "*.CL" file that could not be
interpreted. These items should be checked to ensure valid tape output.

NOTE

The GNC Interface kit requires IXCL runtime license.

For ALL CYCLEs, the control points issued by GNC will be adjusted to match the
ISO definition of CYCLE points. All points will be shifted along their tool
vectors (in the opposite direction) by the clearance amount ( c ).

A CYCLE/OFF will be output before the first command (any post-processor command)
following all CYCLE GOTO points.
