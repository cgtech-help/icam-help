3DEXPERIENCE TO CAM-POST INTERFACE DOCUMENTATION

Overview

The 3DEXPERIENCE Interface Kit is designed to accommodate differences in syntax between Dassault Systèmes and ICAM's post processing languages. Dassault Systèmes APT commands in a cutter location source (".aptsource") file are input to CAD-APT and CAM-POST. See CAM-POST chapter "Post processor Vocabulary" for more information on CAM-POST supported syntax.

The 3DEXPERIENCE Interface Kit sets the input format to "3DEXPERIENCE", sets the input card length to 72 characters, and tells GENER to ignore null parameters (i.e., two commas in succession) in the input aptsource file.

3DEXPERIENCE PPTable

Two PPTable files containing ICAM compatible post processor command syntax are included with the 3DEXPERIENCE interface kit. These files (or similar ones) can also be found in the 3DEXPERIENCE Startup\Manufacturing\PPTables directory, which is where 3DEXPERIENCE expects to find them. If you make changes to a PPTable file, you should be sure to:

  1. Save your changes using a unique file name;
  2. Copy the new PPTable file to 3DEXPERIENCE's Startup\Manufacturing\PPTables directory.

These PPTable files are provided as a starting point only. In practice, each post processor may require its own PPTable file.
